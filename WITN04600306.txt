Jan,

I've spoken to James Stinchcombe on the topic we discussed we spoke about today. It is the case that
transactions held on the session stack but not committed are held in memory only and not written to the
message store until the session is settled.

So any reboots that took place during a session would have wiped all EPOSS transactions. APS, NBS and
ETU all create recovery records which are checked at reboot. Some of these are automatically recovered (and
marked as such within message store). Others will request the clerk to confirm that the transaction was
completed at which point they are committed or deleted.

Hope this helps.
